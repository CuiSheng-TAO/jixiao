# 终评工作台二次修订设计

## 背景

第一版终评工作台已经把原有 `/calibration` 拆成了三标签结构，并具备终评配置、普通员工终评、主管层双人终评等业务能力。但从真实使用视角看，仍然存在明显问题：

- 页面仍偏“系统工作台”，不是“绩效终评决策台”
- 过程信息暴露过多，默认把谁做了什么展示得太细，不符合终评场景
- 多处使用原生 `select` 或长滚动列表，阅读和操作成本高
- 名单范围没有被前置约束，导致“这一轮到底评谁”不够清楚
- 后台配置区域使用原生多选大框，简陋且难懂
- 部分旧页面仍存在小的可用性问题，例如 `self-eval` 主管/管理员预览卡在 loading

本轮修订不违背最初的业务文字要求，重点是把终评页面改得更符合常见绩效系统的思路：先看全局、再看重点、再拍板；过程信息分层可见；配置交互清晰可控。

## 目标

- 保留最初的三标签业务结构：`原则`、`非主管员工终评`、`主管层双人终评`
- 将工作台从“系统过程展示”调整为“支持终评拍板”
- 让大多数使用者先看到结果、风险和待办，而不是先看到细碎过程
- 用更符合直觉的名单选择与配置方式替换原生多选框和下拉框
- 将本轮普通员工终评对象默认收敛到指定的 54 人名单
- 修复终评相关旧问题中暴露出来的可用性 bug

## 非目标

- 不改变最初约定的三标签结构
- 不改变“终评相关人 / 最终确认人 / 主管层双人终评填写人 / 主管层终评名单”这四类配置概念
- 不改变官方最终结果写回现有正式结果字段的方式
- 不引入批量粘贴名单或整批替换名单的配置方式
- 不重做已有初评问卷内容本身

## 设计原则

### 1. 默认看结果，不默认看过程

终评页面的第一职责是帮助拍板，不是展示所有人的操作轨迹。

- 默认先展示分布、风险、待拍板状态、官方结果
- 具名意见和详细过程属于二级信息
- 只有真正需要判断或拍板的人，才展开更细信息

### 2. 页面先回答三个问题

每个终评页面都应优先回答：

1. 现在整体情况怎么样
2. 现在最该处理谁
3. 当前选中这个人应该怎么拍板

如果一个区块不能帮助回答这三个问题，就不应该占据主视觉。

### 3. 配置要像“选人”，不是像“数据库字段”

后台配置区应当服务管理员快速确认：

- 这组名单有多少人
- 这组名单分别是谁
- 这一组和别的组有没有重叠
- 我如何快速加一个人、删一个人

原生多选大框不满足这些要求，必须替换。

## 过程信息可见性

为了避免过程信息被过度公开，终评界面按角色分层展示：

### 普通员工终评

- 普通可访问者：
  - 只看汇总状态
  - 只看分布、待办、是否有分歧、当前官方结果
  - 不看具名意见明细
- 最终确认人 + `ADMIN`：
  - 可以看具名意见明细
  - 可以看谁同意、谁改星、理由是什么

### 主管层双人终评

- 填写人本人：
  - 只可编辑自己的问卷
- 最终确认人 + `ADMIN`：
  - 可看双人具名明细
  - 可做最终确认
- 其他可访问者：
  - 仅看汇总状态与最终结果
  - 不看双人问卷详细内容

## 页面结构修订

## 标签 1：原则

`原则` 页仍然保留“briefing + 总览”的作用，但需要进一步压缩成真正面向高层使用的入口页。

### 上半屏：briefing

只保留四类信息：

- 终评原则
- 角色与链路
- 截止时间与倒计时
- 当前最重要的 2-3 个风险提醒

要求：

- 全部使用短句、人话表达
- 去掉不必要的系统指标解释
- 不把过程统计卡堆成一排相同盒子

### 下半屏：总览

保留两个主图：

- 全公司星级分布图
- 分数带 / 分层图

并在图旁给出一句话解读，例如：

- 哪个档位偏高
- 哪些高分带员工尚未拍板
- 当前最优先处理哪类人

## 标签 2：非主管员工终评

仍采用“左看盘、右拍板”的结构，但把信息进一步收口。

### 左侧：看盘区

只保留三类最有决策价值的信息：

1. 星级分布图
2. 分数带 / 分层图
3. 重点名单

### 星级分布图

- 已最终确认员工按官方星级统计
- 未最终确认员工按参考星级统计
- 图边用简短说明解释当前口径
- 不再堆砌多余指标卡

### 分数带 / 分层图

- 显示初评加权分的大致聚集区
- 帮助理解参考星级落点
- 图上明确标出分段与人数，不做抽象装饰型图表

### 重点名单

以卡片分组而不是大表格：

- 待拍板
- 有分歧
- 高分带未定
- 超线敏感区

点击名单项后，右侧详情面板切换。

### 右侧：决策面板

右侧始终固定为当前选中员工的决策面板，顺序如下：

1. 最终决策区
2. 证据摘要区
3. 汇总态区
4. 过程明细区（按权限展示）

### 最终决策区

这是第一视觉优先区块，展示：

- 姓名、部门、岗位
- 参考星级
- 当前官方星级
- 当前状态
- 最终确认动作

### 证据摘要区

默认只显示帮助拍板的核心证据：

- 初评加权分
- 360 汇总情况
- 自评状态
- 关键评语摘要

不把所有原始文本一次性铺开。

### 汇总态区

默认只展示汇总，不展示具名细节，例如：

- 已有多少位终评相关人处理
- 是否有人提出改星
- 是否仍有待处理意见

### 过程明细区

按权限显示：

- 普通可访问者看不到具名意见
- 最终确认人和 `ADMIN` 可以展开查看具名意见及理由

## 标签 3：主管层双人终评

主管层页继续保持“左看盘、右拍板”，但进一步强调“汇总优先、细节按权限展开”。

### 左侧

保留三块：

- 主管层星级分布
- 双人提交进度
- 主管名单

### 主管名单

使用可搜索、可点击的名单组件，而不是下拉框或简单长列表。

每个对象卡显示：

- 姓名
- 当前状态
- 双人是否齐备
- 是否待拍板

### 右侧

默认结构：

1. 最终决策区
2. 双人汇总对照区
3. 问卷详情区（按权限展开）
4. 确认与留痕区

### 汇总对照区

对于普通可访问者，只显示：

- 两位填写人的提交状态
- 是否已双人齐备
- 当前汇总态

对于填写人本人、最终确认人、`ADMIN`，才可以展开看到具体打分与问卷内容。

## 选择与配置交互

## 工作台内选人

工作台中的“选人”全部改为更符合直觉的方式：

- 不使用原生下拉框
- 使用可搜索名单 + 点击切换详情面板
- 支持按状态过滤
- 每个对象用清晰状态标签标识

## 后台配置页

后台配置页所有名单类配置统一改成：

- 顶部标题 + 当前人数
- 搜索框：搜索并添加成员
- 下方已选成员卡片区

每个已选成员卡片展示：

- 姓名
- 部门
- 角色
- 删除按钮

支持的交互方式仅有：

- 搜索添加
- 删除成员

不支持：

- 原生多选大框
- 批量粘贴名单
- 一键整批替换

### 名单卡片区的附加提示

配置区应提供轻量提示：

- 某人是否同时出现在多个名单中
- 某组名单当前是否为空
- 某组名单是否人数明显异常

这些提示用于帮助管理员检查配置，不打断操作。

## 可视化修订

## 终评工作台中的图

图表必须承担判断职责，不做“为了有图而有图”。

本轮保留的图形类型：

- 星级分布图
- 分数带 / 分层图

要求：

- 图旁必须有一句人话解释
- 必须清楚标出人数或比例
- 必须能看懂当前统计口径
- 不使用装饰性过强但难解释的图形

## 后台配置页中的“图”

后台配置页不再使用类似截图中的大滚动框伪装成图。

配置页本质上是成员管理界面，应当用：

- 搜索
- 标签
- 已选成员卡片
- 数量统计

来承担信息展示，而不是继续用四个原生多选框。

## 兼容与修复

本轮除了终评工作台本身的结构调整，还应顺手修复相关可用性问题：

- 修复 `self-eval` 页面中主管/管理员预览卡在 loading、无法显示内容的问题

## 成功标准

完成后，系统应满足：

- 非终评核心人员不会默认看到过细的具名过程信息
- 使用者进入页面后，能在短时间内看懂：
  - 当前整体情况
  - 当前最该处理的人
  - 当前选中对象如何拍板
- 普通员工终评默认只围绕指定 54 人展开
- 后台名单配置从“原生多选大框”改为“搜索添加 / 删除”的成员管理模式
- 页面中的选人交互不再依赖下拉框
- 图表比当前版本更能支持判断，而不是制造阅读成本
