# 绩效考核系统 - 飞书接入说明

## 概述

本系统使用 **飞书自建应用** 实现 OAuth 登录，同时通过飞书开放 API 获取组织架构（部门/成员）和发送消息通知。

---

## 前置工作清单

### 第一步：创建飞书自建应用

1. 打开 [飞书开放平台](https://open.feishu.cn/app) ，使用管理员账号登录
2. 点击 **创建自建应用**
3. 填写应用名称（如"绩效考核系统"）和描述
4. 创建完成后，在应用的 **凭证与基础信息** 页面获取：
   - `App ID` → 对应环境变量 `FEISHU_APP_ID`
   - `App Secret` → 对应环境变量 `FEISHU_APP_SECRET`

### 第二步：配置应用权限（数据范围）

在应用的 **权限管理** 页面，开启以下权限：

| 权限名称 | 权限标识 | 用途 |
|---------|---------|------|
| 获取用户基本信息 | `contact:user.base:readonly` | 登录时获取用户姓名、邮箱、头像 |
| 获取用户 open_id | `authen:user.id:readonly` | 登录时获取用户唯一标识 |
| 获取部门基本信息 | `contact:department.base:readonly` | 同步组织架构（部门列表） |
| 获取部门成员列表 | `contact:user.employee_id:readonly` | 同步部门下的成员 |
| 发送消息 | `im:message:send_as_bot` | 发送绩效通知卡片消息 |

> 注意：部分权限需要 **管理员审批** 才能生效，请提前申请。

### 第三步：配置网页登录（重定向地址）

1. 在应用的 **安全设置** 页面
2. 添加 **重定向 URL**：
   - 本地开发：`http://localhost:3000/login`
   - 生产环境：`https://你的域名/login`

### 第四步：发布应用

1. 在 **版本管理与发布** 中创建版本
2. 提交审核 → 管理员审批通过后，应用才对组织内员工可用
3. 如果只是测试，可以先在 **可用范围** 中设置为仅部分人可用

## 环境变量配置

复制 `.env.example` 为 `.env`，填入以下内容：

```env
# 飞书应用凭证（从开放平台获取）
FEISHU_APP_ID="cli_xxxxxxxxxxxxxx"
FEISHU_APP_SECRET="xxxxxxxxxxxxxxxxxxxxxxxx"

# 前端需要的 App ID（用于拼接 OAuth 跳转链接）
NEXT_PUBLIC_FEISHU_APP_ID="cli_xxxxxxxxxxxxxx"

# NextAuth 密钥（随机字符串，用于加密 session）
AUTH_SECRET="随机生成一个长字符串"

# 应用访问地址
AUTH_URL="http://localhost:3000"
NEXT_PUBLIC_APP_URL="http://localhost:3000"
```

## 登录流程说明

```
用户点击"飞书登录"
       |
       v
跳转飞书 OAuth 授权页面
(open.feishu.cn/open-apis/authen/v1/authorize)
       |
       v
用户在飞书中确认授权
       |
       v
飞书回调到 /login?code=xxx
       |
       v
前端将 code 发送到 /api/auth/feishu-login
       |
       v
后端用 code 换取 user_access_token
       |
       v
后端用 token 获取用户信息（姓名、邮箱、open_id）
       |
       v
在数据库中创建/更新用户记录
       |
       v
生成 NextAuth session cookie，完成登录
```

## 用到的飞书 API 列表

| API | 用途 |
|-----|------|
| `POST /auth/v3/tenant_access_token/internal` | 获取应用级别的 tenant_access_token |
| `POST /authen/v1/oidc/access_token` | 用授权码换取用户 token |
| `GET /authen/v1/user_info` | 获取当前登录用户信息 |
| `GET /contact/v3/departments` | 获取部门列表 |
| `GET /contact/v3/users` | 获取部门下的成员列表 |
| `POST /im/v1/messages` | 发送消息（绩效通知卡片） |

## 常见问题

**Q: 点击飞书登录没有反应？**
A: 检查 `NEXT_PUBLIC_FEISHU_APP_ID` 是否配置。如果没配置，系统会回退到开发模式的演示登录。

**Q: 登录报错 "Failed to get user token"？**
A: 常见原因：重定向 URL 没有在飞书开放平台配置，或 App Secret 填错了。

**Q: 获取部门/成员失败？**
A: 检查对应的通讯录权限是否已申请并通过审批。

**Q: 消息发送失败？**
A: 确认 `im:message:send_as_bot` 权限已开启，且目标用户在应用的可用范围内。
